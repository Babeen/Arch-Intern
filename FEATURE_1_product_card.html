Overview — What We Built

We added three improvements to every product card:

  1. Quick View Modal — a popup that shows product details without leaving the page
  2. Image Hover Swap — a second image fades in when the user hovers over the card
  3. Size Selector Pills — S / M / L / XL buttons on the card itself

These are standard features on every major fashion e-commerce site (Zara, ASOS, Adidas). They reduce friction — the user gets more information without having to navigate away and come back.

Part 1 — Concept: useState

useState is a React Hook that lets a component remember a value between renders. Every time the value changes, React re-renders the component to show the updated UI.

Syntax

const [value, setValue] = useState(initialValue);
  • value — the current value you can read
  • setValue — the function you call to change it
  • initialValue — what the value starts as (false, null, "", 0, etc.)

How we use it in ProductCard

// Controls whether the Quick View modal is open or closed
const [showModal, setShowModal] = useState(false);

// Tracks which size pill the user clicked (null = none selected)
const [selectedSize, setSelectedSize] = useState(null);
Mental model: Think of useState like a light switch. The switch has a current position (on/off = true/false). You can read the position (showModal) and you can flip it (setShowModal(true)). Every time you flip it, React re-draws the component.

Part 2 — Concept: Event Bubbling & stopPropagation

This is one of the most important JavaScript concepts for building interactive UIs.

What is event bubbling?

When you click an element, the click event doesn't just fire on that element — it "bubbles up" through every parent element all the way to the top of the page. Each parent's onClick also fires.

// Imagine this structure:
<div onClick={() => console.log("DIV clicked")}>
  <button onClick={() => console.log("BUTTON clicked")}>
    Click me
  </button>
</div>

// When you click the button, BOTH fire:
// → "BUTTON clicked"
// → "DIV clicked"   ← this is bubbling

Why this matters in our card

The product card has a <Link> wrapping the image area. If we put a "Quick View" button inside that area, clicking Quick View would ALSO trigger the Link and navigate to the product page — not what we want.

// Without stopPropagation — clicking Quick View navigates away (BAD)
<button onClick={() => setShowModal(true)}>Quick View</button>

// With stopPropagation — click stays on the button only (GOOD)
<button onClick={(e) => {
  e.stopPropagation(); // stop the click from reaching the Link
  setShowModal(true);
}}>Quick View</button>
Common mistake: Forgetting stopPropagation on buttons inside links or cards. The symptom is: clicking a button also triggers an unintended navigation or parent click handler.

Part 3 — Concept: CSS group / group-hover (Tailwind)

Tailwind's group utility lets child elements respond to their parent's hover state. This is how we show the Quick View button only when the card is hovered.

How it works

// Step 1: Add `group` to the PARENT element
<div className="group bg-white rounded-2xl">

  // Step 2: Use `group-hover:` on any CHILD element
  <button className="opacity-0 group-hover:opacity-100 transition">
    Quick View
  </button>

</div>

// Result: button is invisible normally, visible when the div is hovered

The "reveal from below" effect on Quick View button

className="
  opacity-0 translate-y-2        ← default: invisible, 8px below
  group-hover:opacity-100        ← on hover: visible
  group-hover:translate-y-0      ← on hover: moves to normal position
  transition-all duration-300    ← smooth animation between states
"
Mental model: group is like a "zone". Any element inside the zone can say "when the zone is hovered, change me". Without group, a child element can only react to being hovered itself, not its parent.

Part 4 — Image Hover Swap

ProductCard.jsx — Image Area

How it works

Both images are placed in the same container using absolute positioning — they stack on top of each other. The primary image is visible by default. The hover image is invisible (opacity-0) and slightly scaled down (scale-95). On hover, they swap.

// Container — `relative` so children can use `absolute`
<div className="relative h-64 overflow-hidden">

  // Image 1 (primary) — visible, fades OUT on hover
  <img className="absolute opacity-100 scale-100
                   group-hover:opacity-0 group-hover:scale-95
                   transition-all duration-700" />

  // Image 2 (hover) — invisible, fades IN on hover
  <img className="absolute opacity-0 scale-95
                   group-hover:opacity-100 group-hover:scale-100
                   transition-all duration-700" />

</div>
Real project note: The Fake Store API only provides one image per product. We simulate the second image by appending ?v=2 to the URL (which loads the same image — just to show the mechanism works). In a real backend, you'd store multiple images per product and use product.images[1] here.

Part 5 — Size Selector Pills

ProductCard.jsx — Size Pills

How it works

We define a SIZES array outside the component (so it's not recreated on every render). We map over it to render a button for each size. The selected size gets a dark filled style; others get a bordered style.

// Defined OUTSIDE the component — constant, never changes
const SIZES = ["S", "M", "L", "XL"];

// Inside the component:
{SIZES.map((size) => (
  <button
    key={size}
    onClick={(e) => {
      e.preventDefault();   // don't follow any link
      e.stopPropagation(); // don't bubble to parent
      // Toggle: clicking the same size again deselects it
      setSelectedSize(selectedSize === size ? null : size);
    }}
    className={selectedSize === size ? "bg-gray-900 text-white" : "border text-gray-500"}
  >
    {size}
  </button>
))}
Why e.preventDefault()? If a size button is inside a <Link> or <form>, the default browser behavior would navigate or submit. preventDefault() cancels that default behavior. stopPropagation() stops the event from reaching parent elements. They do different things — you often need both.

Concept: useEffect for side effects

A "side effect" is anything that affects something outside the component — like adding an event listener, changing the document title, or modifying document.body.

// Side effect 1: Close modal when Escape key is pressed
useEffect(() => {
  const handleKey = (e) => {
    if (e.key === "Escape") onClose();
  };

  window.addEventListener("keydown", handleKey);

  // CLEANUP FUNCTION — runs when the component is removed from the DOM
  // Without this, the listener keeps running even after the modal closes
  return () => window.removeEventListener("keydown", handleKey);

}, [onClose]); // dependency array — re-runs if onClose changes


// Side effect 2: Prevent background scroll while modal is open
useEffect(() => {
  document.body.style.overflow = "hidden"; // lock scroll
  return () => { document.body.style.overflow = ""; }; // restore on close
}, []);
The dependency array [ ]
The second argument to useEffect controls WHEN it re-runs:
useEffect(fn, []) — runs once when the component mounts (appears)
useEffect(fn, [value]) — runs when value changes
useEffect(fn) — runs after every render (usually not what you want)

Concept: AnimatePresence

React normally removes elements from the DOM instantly when a condition becomes false. Framer Motion's AnimatePresence intercepts this removal and lets the element play its exit animation first.

// In ProductCard.jsx:
<AnimatePresence>
  {showModal && (
    <QuickViewModal
      key="quick-view"   ← key tells AnimatePresence which element this is
      product={product}
      onClose={() => setShowModal(false)}
    />
  )}
</AnimatePresence>

// In QuickViewModal.jsx — the exit animation:
<motion.div
  initial={{ opacity: 0, scale: 0.95, y: 20 }}  ← starting state
  animate={{ opacity: 1, scale: 1, y: 0 }}       ← visible state
  exit={{ opacity: 0, scale: 0.95, y: 20 }}      ← exit state (plays before removal)
/>
Why does the modal need a key prop? AnimatePresence uses the key to identify which child is entering/exiting. Without it, it may not detect the change correctly. Always give a unique key to direct children of AnimatePresence.

The "Added to Cart!" feedback

const [added, setAdded] = useState(false);

const handleAdd = () => {
  addToCart({ ...product, size: selectedSize });
  setAdded(true);
  setTimeout(() => setAdded(false), 1500); // reset after 1.5 seconds
};

// In JSX — button text and color change based on `added`:
className={added ? "bg-green-500" : "bg-gray-900 hover:bg-amber-400"}
{added ? "Added to Cart!" : "Add to Cart"}
