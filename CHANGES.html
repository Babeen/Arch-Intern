Frontend UI — Changes Report

Project: Arch-Intern Frontend  |  Stack: React + Vite + Tailwind CSS v4 + Framer Motion  |  Prepared by: Amazon Q

1. Overview

This document covers all UI changes made across three separate sessions:

  • Session 1: Home page premium upgrade — full-viewport hero, transparent navbar, Framer Motion animations.
  • Session 2: Consistent premium theme applied to all pages + new reusable components created.
  • Session 3: Navbar components updated — all blue colors removed, amber/dark theme applied throughout.

The core design language established is: dark neutrals (gray-900) as the primary action color, amber-400/500 as the accent, white/light surfaces for cards, and rounded-full pill shapes for all buttons. Blue was completely removed from the entire codebase.

2. Session 1 — Home Page & Layout Overhaul

src/assets/layouts/MainLayout.jsx CHANGED

What changed

  • Removed the hardcoded max-w-7xl mx-auto px-6 py-10 wrapper from <main> so the hero section on the Home page can stretch edge-to-edge (full bleed).
  • Added a transparentNav prop. When passed as true, it forwards a transparent prop to the Navbar, enabling the see-through navbar effect on the Home page.
  • Changed background from bg-gray-100 to bg-white for a cleaner base.

Why

The old layout forced padding on every page, which prevented the hero image from covering the full screen. Decoupling the padding lets each page control its own spacing via PageWrapper.

src/assets/components/layout/Navbar.jsx CHANGED

What changed

  • Changed from sticky to fixed positioning so it overlays the hero image instead of pushing content down.
  • Added a transparent prop and a scroll listener using useEffect. When the user is at the top of the page and transparent=true, the navbar shows as bg-white/10 backdrop-blur-sm (low opacity glass effect). Once the user scrolls past 60px, it transitions to a solid bg-white/95 backdrop-blur-md shadow-md.
  • All text, icons, and buttons dynamically switch color based on the transparent state — white when over the hero, dark when solid.
  • Blue removed: Logo changed from text-blue-600 to text-gray-900 dark:text-white.
  • Blue removed: Cart badge changed from bg-blue-600 text-white to bg-amber-400 text-gray-900.
  • Blue removed: Signup button replaced — the old blue Button component was swapped for a plain Link styled as a dark pill with hover:bg-amber-400.
  • Removed the unused Button and Moon/Sun imports.

src/assets/pages/Home.jsx CHANGED

What changed

  • Hero image fix: The background image was previously set via a relative string path in CSS (url('../../assets/images/home.png')) which Vite does not process. It now uses a proper ES module import (import heroBg from "../images/home.png") so Vite handles it correctly.
  • Full viewport hero: Changed from min-h-[90vh] to h-screen so the hero covers the entire screen including behind the navbar.
  • Ken Burns animation: The background image starts slightly zoomed in (scale: 1.12) and animates to scale: 1.02 on load using Framer Motion, giving a cinematic feel.
  • Scroll indicator: Added an animated pulsing vertical line with a "Scroll" label at the bottom of the hero.
  • Typography upgrade: Hero headline uses a mixed bold + italic weight ("Urban Edge" in light italic) for a premium editorial look.
  • Amber accent labels: Every section now has a small uppercase amber label above the main heading (e.g., "New Arrivals — SS 2025", "Collections", "Trending").
  • Framer Motion animations: All sections use whileInView with staggered fadeUp variants so elements animate in as the user scrolls.
  • Blue removed: Product prices changed from text-blue-600 to text-gray-900. All buttons changed to dark pill style with amber hover.
  • Newsletter section: Replaced the blue gradient background with a dark bg-gray-950 card featuring decorative blurred amber and blue blob shapes for depth.
  • Removed the heroLoaded state and useEffect preloader — no longer needed since the image is imported directly.
